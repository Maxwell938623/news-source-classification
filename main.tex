\section{Introduction}
In this project, we built an end-to-end binary NLP classification pipeline to predict whether a headline came from Fox News or NBC News. Our core objective was to make the system reproducible and extensible: every major stage (URL collection, scraping, preprocessing, splitting, training, evaluation, and prediction) is implemented as a script.

We first reproduced a simple reference baseline (TF-IDF capped at 100 terms plus logistic regression with a 100-iteration limit), then expanded to a controlled experiment framework covering multiple feature and model families. Our training framework evaluates a large configuration space and selects the best model by validation macro-F1 before retraining on train+validation and evaluating once on the held-out test set.

This report reflects the finalized implementation details and experimental outcomes produced.

\section{Core Components}

\subsection{Data Collection and Dataset}
Our data pipeline follows a two-stage collection process designed to maximize coverage while keeping source labels reliable.
\begin{itemize}
    \item \textbf{URL discovery.} A dedicated collector traverses robots-discovered sitemap indexes, configured sitemap seeds, RSS/Atom feeds, and section pages for FoxNews and NBC domains. We avoid relying on a single source (for example only one sitemap) because that misses many valid article pages, especially for historical coverage. To increase size and coverage, we ran the data scraper continuously for seven consecutive days. Collection stays within publicly advertised article URLs for the two outlets, respects robots.txt where applicable, uses modest concurrency, and pairs exponential backoff with retries so we do not hammer hosts when pages fail transiently.

    \item \textbf{Headline scraping.} The headline scraper fetches each URL with retry/backoff logic and applies a robust extraction cascade so template variation across pages does not cause widespread failures:
    \begin{enumerate}
        \item source-specific heading-tag class heuristics,
        \item Open Graph and schema.org headline fields where present,
        \item first generic page heading,
        \item Open Graph title,
        \item fallback document title.
    \end{enumerate}
\end{itemize}

At the URL-discovery stage we have 172,488 discovered URLs (115,266 FoxNews; 57,222 NBC). Each row carries a binary label from the publisher associated with the URL (FoxNews versus NBC); preprocessing drops rows with missing labels, conflicting labels, or failed scrapes. Preprocessing enforces scrape-status filtering, null/empty removal, HTML cleanup, whitespace normalization, within-source deduplication, cross-source duplicate removal, and label validation. After cleaning, the merged scrape pool contains 114,306 successfully scraped headlines.

For the classical-model experiments we built a 20,000-headline cleaned dataset: we keep the provided helper CSV as the anchor set and draw a fixed-seed random sample from the larger scrape, excluding duplicate helper URLs and headlines and targeting an approximately balanced source mix. The cleaned table has 10,513 FoxNews headlines (class 0) and 9,487 NBC headlines (class 1), with mean headline length about 13.2 words on minimally normalized text. Stratified 70/15/15 splits use seed 42 and yield 13,999 training examples, 3,000 validation examples, and 3,001 held-out test examples. The split is random within each class rather than chronological: train, validation, and test all draw from the same underlying time range, which simplifies variance estimation but means topical drift toward the leaderboard deadline is handled only indirectly through the limitations summarized under Error Analysis and Limitations.

\subsection{Model Design}
We compared sparse classical models, dense embedding heads, ensemble combinations, and fine-tuned transformer classifiers on the shared headline splits.

Training protocol (classical and embedding heads): FoxNews encoded as class 0 and NBC as class 1; stratified splits use seed 42; hyperparameters are chosen by validation macro-F1; the selected model is refit on train plus validation and scored once on the held-out test set. \emph{Note on label encoding:} the handout's §3.3 baseline snippet contains an internal contradiction --- the comment specifies ``0 for FoxNews, 1 for NBC'' while the lambda code on the next line uses the opposite mapping. We follow the comment because (a) it matches the implicit ordering in the URL CSV and (b) the leaderboard backend documents a robust 2-class remapping in §4.2 of \texttt{submission.txt}, so the reverse encoding would still be scored correctly.

Baseline:
\begin{itemize}
    \item TF-IDF (100 features) with logistic regression.
\end{itemize}

Classical and dense families:
\begin{itemize}
    \item Word TF-IDF with logistic regression, linear SVM, or multinomial naive Bayes.
    \item Character n-gram TF-IDF with linear models (including fused word--character pipelines).
    \item Stylometric features with logistic regression, SVM, random forest, or histogram gradient boosting.
    \item Hybrid sparse inputs combining word TF-IDF, character TF-IDF, and stylometry.
    \item Soft-voting ensembles over heterogeneous calibrated bases.
    \item Stacking ensembles with a meta-classifier on base-model probabilities.
    \item Gradient boosted trees built from sparse text-derived features (TF-IDF plus stylometry).
    \item Sentence-transformer embeddings with linear, MLP, LightGBM, or XGBoost heads.
    \item Combined word--character TF-IDF with linear SVM; supervised fastText; averaged static embeddings with a classifier head.
\end{itemize}

Fine-tuned transformers (SOTA checkpoints adapted to headline classification):
\begin{itemize}
    \item DistilBERT-base uncased.
    \item DeBERTa-v3-small.
    \item ModernBERT-base (AnswerDotAI).
\end{itemize}

Submitted artifacts. We packaged four leaderboard-ready submissions under \texttt{submission/}, each conforming to the \texttt{model.py} + \texttt{preprocess.py} + \texttt{model.pt} contract in \texttt{submission.txt}, so the team can swap the active package between leaderboard attempts based on observed accuracy:
\begin{itemize}
    \item \texttt{submission/distilbert\_model/} --- fine-tuned DistilBERT-base uncased (highest test macro-F1 in Table~\ref{tab:results}).
    \item \texttt{submission/modernbert\_model/} --- fine-tuned ModernBERT-base sequence classifier.
    \item \texttt{submission/stack\_4base\_histgbm\_model/} --- four-base stacking ensemble with histogram gradient boosting meta-learner (strongest sparse pipeline in Table~\ref{tab:results}); serialised as a scikit-learn \texttt{Pipeline} embedded inside a torch \texttt{state\_dict} via a byte-buffer wrapper so the leaderboard backend's \texttt{torch.load} machinery can transparently reconstruct it.
    \item \texttt{submission/gbdt\_model/} --- gradient boosted trees over TF-IDF + stylometry features (compact baseline reference).
\end{itemize}

\subsection{Evaluation and Model Performance}
We report accuracy, macro-F1, per-class F1, confusion matrices, and (when available) ROC summaries as diagnostics; model selection still tracks validation macro-F1.

Held-out accuracy and macro-F1 across models are summarized in Table~\ref{tab:results}. Validation macro-F1 for selection was 0.8004 for the HistGBM-meta stack, 0.7727 for the GBDT hybrid, and 0.8218 for ModernBERT (other validation scores omitted here). The table omits per-class F1; on the headline split those are, for example, baseline FoxNews 0.6736 and NBC 0.5588; HistGBM-meta stack 0.8018 and 0.7893; GBDT hybrid 0.7721 and 0.7518; ModernBERT 0.8284 and 0.8078 on its 3{,}000-example fold. Supplementary plots include confusion matrices for the baseline and strong checkpoints and validation macro-F1 across classical configurations (Figure~\ref{fig:results_visual}).

Relative to baseline on the 20k split, the HistGBM-meta stack lifts test accuracy by 17.1 percentage points (from 0.6248 to 0.7957) and macro-F1 by 17.9 points (from 0.6162 to 0.7955). DistilBERT adds roughly another 3.1 macro-F1 points on top of that stack. ModernBERT-base reaches 0.8181 macro-F1 on its held-out fold, slightly below DistilBERT and DeBERTa on this split but still competitive.

\subsection{Error Analysis and Limitations}
\begin{itemize}
    \item \textbf{Research question and hypothesis.} We hypothesized that remaining errors would concentrate on linguistically shallow headlines (few tokens, generic ledes) and on politically overloaded wording shared by both outlets. Those patterns should show up as symmetric confusion off-diagonals for strong balanced models and as low-confidence mistakes in sparse pipelines.

    \item \textbf{Methods and evidence.} We quantify mistakes using held-out confusion matrices for the HistGBM stacking model, ModernBERT, and GBDT (Table~\ref{tab:results}). Figure~\ref{fig:experiment_comparison} summarizes classical validation macro-F1 across experiment configurations; Figure~\ref{fig:modernbert_cm} shows ModernBERT's confusion matrix on its held-out fold.

    \item \textbf{Primary findings (per-class breakdown).}
    \begin{itemize}
        \item 20k headline split ($N = 3{,}001$ test; 1,577 Fox / 1,424 NBC): the best classical stacking model misclassifies 337 Fox-as-NBC (21.4\%) and 276 NBC-as-Fox (19.4\%). The balanced source sample reduces the extreme class-asymmetry seen in earlier larger-corpus runs, so the remaining errors are closer to symmetric and are driven more by headline ambiguity than by class prior alone.
        \item ModernBERT held-out fold (3{,}000 examples; 1{,}577 Fox / 1{,}423 NBC): 264 Fox-as-NBC (16.8\%) and 280 NBC-as-Fox (19.7\%), broadly comparable asymmetry to the classical stack at similar class supports.
        \item GBDT hybrid on the headline split ($N = 3{,}001$): Fox-as-NBC errors are 369 of 1{,}577 (23.4\%) and NBC-as-Fox errors are 344 of 1{,}424 (24.2\%).
    \end{itemize}

    \item \textbf{Failure modes (qualitative review).} Short or generic headlines show up often among uncertain mistakes because they carry little lexical or style signal. Politics-heavy headlines share vocabulary across outlets (for example mentions of major candidates or elections), where calibrated sparse models lose confidence. Slug-style fallbacks still hurt when a leaderboard URL cannot be scraped cleanly.

    \item \textbf{Limitations.}
    \begin{itemize}
        \item The final 20k dataset is one random draw from a larger scrape, so coverage is broad but not exhaustive.
        \item Our local held-out test predates the leaderboard release; topic drift after the deadline will change behaviour in ways we cannot measure here.
        \item Transformer inputs use fixed maximum sequence lengths during fine-tuning, which clips rare long headlines.
        \item We fine-tune on headlines only; full article bodies are outside the scope of this benchmark.
    \end{itemize}
\end{itemize}

\section{Exploratory Components}

Beyond the reference baseline we documented several exploratory threads that shaped reliability and modeling decisions.

\subsection*{Code and engineering}
\begin{itemize}
    \item \textbf{Motivation.} Scraping news at scale breaks unless ingestion is restartable, observable, and gentle on upstream sites.

    \item \textbf{Approach.} We added a continuous scrape driver that chains URL discovery, CSV merges, and resumed headline extraction with structured logs so nightly jobs replace manual copy-paste workflows. The headline scraper combines templated heuristics because Fox and NBC pages diverge in markup; layering Open Graph and HTML fallbacks reduced blank-headline rates compared with trusting a single DOM selector.

    \item \textbf{Challenges and mitigations.} Duplicate URLs reappear across sitemap refreshes, so merge logic deduplicates on canonical URLs before labeling. Transient HTTP failures formerly stalled batches; exponential backoff plus capped parallelism stabilized week-long runs without hammering hosts.

    \item \textbf{Impact.} Qualitatively, the pipeline raised usable scraped rows to six figures and kept collection reproducible enough to regenerate the twenty-thousand headline study split from the same scripts; quantitative throughput varies with network conditions, but structured logs let us verify cycle completion rather than guessing whether a manual run finished.
\end{itemize}

\subsection*{Techniques and literature grounding}
\begin{itemize}
    \item \textbf{Sparse feature families.} Word TF-IDF baselines follow textbook sparse text classification: bag-of-ngrams with sublinear TF emphasize discriminative tokens without dense storage. Character n-grams follow intuitions from noisy-channel spelling models and SEO-heavy headlines where morphology survives even when lemmas do not. Stylometric features encode shallow writing statistics motivated by authorship-attribution work but tuned here for outlet-level cues such as punctuation balance and casing.

    \item \textbf{Stacking and voting.} We borrow the standard recipe that diversified errors combine when base learners see different input transforms (word versus character versus stylometry); meta-learners trained on out-of-fold probabilities approximate Wolpert-style stacking without leaking validation folds into selection.

    \item \textbf{Dense and transformer upgrades.} Dense sentence embeddings trade interpretability for contextual similarity and proved necessary once hybrid stacks plateaued; transformer fine-tuning upgrades again by conditioning on bidirectional context for ambiguous political wording.

    \item \textbf{Validation choices.} Macro-F1 rewards balanced Fox versus NBC performance under slight remaining imbalance; calibrated SVM heads feed ensembles because raw margins lack probabilities.
\end{itemize}

\subsection*{Dataset expansion}
\begin{itemize}
    \item We experimented with backfill states keyed per outlet so historical URL lists grow outward without repeatedly scraping only the freshest RSS window. That exploratory direction pairs with the continuous scrape loop above and reinforces reproducibility: every enlargement flows through the same preprocessing contract described under Data Collection and Dataset.
\end{itemize}

\subsection*{Analysis thread}
\begin{itemize}
    \item Error Analysis and Limitations formalizes the quantitative outcome of this exploratory thread (hypothesis, evidence, findings). In short, errors concentrate where lexical overlap dominates stylistic signal, while transformers shrink---but do not erase---those pockets of ambiguity.
\end{itemize}

\subsection*{Comparison: sparse pipelines versus contextual encoders}
\begin{itemize}
    \item \textbf{Sparse pipelines.} They remain attractive for latency and debugging: TF-IDF coefficients highlight influential n-grams, and stacking exposes when stylometry compensates for vocabulary overlap. Their downside is brittleness when outlets echo identical entities on breaking news; calibrated probabilities collapse toward half when headlines differ only by function words.

    \item \textbf{Transformer checkpoints.} Contextual embeddings lift accuracy on ambiguous political wording (Table~\ref{tab:results}) but need heavier deployment than sparse pipelines; DistilBERT leads our headline split while ModernBERT remains competitive.

    \item \textbf{Empirical takeaway.} Transformer classifiers outperform our strongest handcrafted pipeline by roughly three macro-F1 points on this snapshot; classical stacks remain useful fallbacks when sparse inference is preferred.
\end{itemize}
